# PRD: 찬반 설문조사 플랫폼

## 1. 제품 개요

### 1.1 제품명
**VotePulse** (가칭) - 실시간 찬반 투표 & 설문 플랫폼

### 1.2 비전
전화 설문조사 중심의 한국 리서치 시장을 온라인으로 전환하고, 누구나 쉽게 설문을 만들고 참여할 수 있는 프로슈머 기반 플랫폼 구축

### 1.3 핵심 가치
- **초간단 참여**: 스와이프/탭 한 번으로 투표 완료
- **프로슈머**: 누구나 설문 등록자이자 응답자
- **속성 기반 인사이트**: 익명 + 프로필 제공자 2트랙 결과 분석
- **재미**: 결과 보는 재미, 내 의견이 다수인지 소수인지 확인

### 1.4 타겟 시장
- **지역**: 대한민국 only (MVP)
- **사용자**: 
  - 응답자: 시간 때우기, 의견 표출, 소소한 재미를 원하는 일반 사용자
  - 등록자: 대학원생, 스타트업, 동아리, 소상공인, 일반 사용자

---

## 2. 사용자 페르소나

### 페르소나 1: 캐주얼 응답자 (민지, 27세)
- 출퇴근 시간에 스마트폰으로 시간 때우기
- "내 생각이 다수인지 궁금해"
- 가볍게 찬반 투표하고 결과 보는 재미

### 페르소나 2: 프로슈머 (준호, 32세)
- 직장에서 간단한 의견 수렴 필요
- "팀원들 점심 메뉴 투표" 같은 가벼운 설문
- 결과를 링크로 공유하고 싶음

### 페르소나 3: 리서처 (수진, 25세, 대학원생)
- 졸업논문용 설문 데이터 필요
- "20대 여성 대상 소비 패턴 조사"
- 빠르고 저렴한 타겟 데이터 수집

## 4. 비기능 요구사항

### 4.1 성능
- 동시 접속 10,000명 이상 처리
- 투표 응답 시간 200ms 이내
- 결과 갱신 실시간 (1초 이내)

### 4.2 확장성
- 일일 방문자 수십만 명 대응 가능한 아키텍처
- 서버리스 기반 자동 스케일링

### 4.3 보안
- 개인정보보호법 준수
- 민감정보 별도 동의/암호화
- 중복 투표 방지 (기기 지문 + IP)

### 4.4 가용성
- 99.9% uptime 목표
- 글로벌 CDN으로 빠른 응답

## 5. 개인정보 설계 원칙

### 5.1 핵심 원칙
> "프로필을 공개하면 혜택"이 아니라, **"프로필 속성을 '통계 처리'에 제공하면 혜택"**

### 5.2 권한 레벨
각 프로필 항목에 대해 사용자가 선택:
1. **나만 보기** - 매칭에만 사용 (설문이 나에게 뜨는 용도)
2. **통계 분석 제공** - 집계/교차분석에만 사용 (개별값 노출 X)

### 5.3 결과 제공 규칙
- 설문 등록자에게 개별 응답 + 개별 프로필 연결 제공 안 함
- 교차분석은 최소 표본수(k명) 이상일 때만 노출
- 익명 응답 / 프로필 제공 응답 분리 표시
